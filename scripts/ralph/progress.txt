**Files changed:**

- src/components/windgram/thermal-cloud-utils.test.ts (new file)
- src/components/windgram/windgram-chart.tsx
- src/components/windgram/index.ts

**Learnings for future iterations:**
- CAPE (Convective Available Potential Energy) is already in the surface data from Open-Meteo API
- The simplified formula W* ≈ 0.12 * sqrt(CAPE) is widely used in soaring forecasts and gives good results
- More complex formulas exist using boundary layer height and heat flux, but require data we don't have
- The thermal velocity should vary through the day: 0 early morning, peak in afternoon, 0 by evening
- When changing function signatures, always update all call sites AND the export/index files
- Unit tests should cover edge cases: null values, zero/negative CAPE, realistic ranges
