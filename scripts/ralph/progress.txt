**Learnings for future iterations:**
- **Lapse rate is the key metric:** Each layer between pressure levels has its own lapse rate. The existing weather-profile.ts only computes ONE lapse rate (850-700 hPa), but windgrams need lapse rates for ALL layers.
- **Color psychology for pilots:** Warm colors (orange/yellow) = unstable air = good thermals. Cool colors (blue/purple) = stable air = poor lift. This matches pilot intuition.
- **Bilinear interpolation for smooth gradients:** Without interpolation, the windgram looks blocky. Sub-cell rendering (2×2 samples per grid cell) creates smooth color transitions while keeping performance good.
- **Handle missing data gracefully:** Bilinear interpolation can fail if any corner is null. Fall back to nearest non-null neighbor to avoid blank cells.
- **Dry adiabatic lapse rate is ~3°C/1000ft:** This is the threshold for unstable air. Anything above this creates thermals.
- **Theme-aware color mapping:** Same lapse rate needs different RGB values in light vs dark mode for good contrast and readability.

**Learnings for future iterations:**
- **Standard meteorological wind barb convention:** Shaft points INTO the direction wind is blowing FROM (meteorological convention). Barbs are drawn on the right side of the shaft (clockwise). Speed rounded to nearest 5 kt: 50kt = flag (triangle), 10kt = full barb, 5kt = half barb.
- **Canvas-based barb rendering:** Drawing barbs directly on canvas (vs SVG overlay) keeps everything in one rendering context, avoiding z-index complexity and simplifying interaction detection.
- **Mobile barb density:** On mobile, showing every barb creates visual clutter. Thinning to every other level AND hour (step=2) reduces density by 75% while maintaining readability.
- **Proximity-based hover:** Instead of checking exact pixel overlap with complex barb shapes, use circular proximity detection (15-20px radius). This is touch-friendly and works well for barbs.
- **Speed category colors match pilot intuition:** Green = light winds = safe flying. Yellow/orange = moderate/strong = caution. Red = dangerous = no-go. This aligns with existing pilot weather conventions.
- **Tooltip formatting for non-technical pilots:** Show both imperial (mph, feet) and metric (kt, meters) units. Include cardinal directions (N, NE, etc.) which are easier to understand than raw degrees.
- **Knot conversion:** 1 kt = 1.852 km/h, so km/h to knots = km/h × 0.539957. Always round to nearest integer for display.
