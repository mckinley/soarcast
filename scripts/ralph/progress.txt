## Codebase Patterns
- `calculateThermalVelocity` canonical location is `src/lib/weather-profile.ts`; re-exported from `thermal-cloud-utils.ts` for windgram consumers
- Open-Meteo returns times in local timezone when `timezone=auto` — extract local hour directly from time string with `time.slice(11, 13)`
- DerivedParameters in `weather-profile.ts` holds all computed atmospheric fields; add new ones there and wire up in the `data.hourly.time.map()` loop inside `fetchAtmosphericProfile`
- Pressure levels array is ordered by PRESSURE_LEVELS constant (1000→500 hPa, i.e., lowest altitude first)
- Scoring v2 uses `calculateDailyScoresFromProfile(profile, forecast, site)` — callers should try v2 with fallback to v1 `calculateDailyScores`
- DayScore has optional v2 fields (wStar, bestWindow, odRisk, windShear, freezingConcern, peakCeilingMsl) — always check for undefined
