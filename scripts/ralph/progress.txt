**Learnings for future iterations:**
- React's useState() only uses initial value on FIRST render - subsequent prop/memo changes don't update state
- The correct pattern for "resetting state when data changes" is using a key prop on the component
- Using fetchedAt timestamp as key is perfect for this - unique per data fetch, forces remount on new data
- ESLint rule react-hooks/set-state-in-effect catches the anti-pattern of calling setState in useEffect
- Better to derive initial state from useMemo and use component key than useEffect with setState
- The component key approach is cleaner and follows React best practices for state initialization
