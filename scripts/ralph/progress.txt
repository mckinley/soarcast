Files changed:

- src/components/windgram/windgram-thumbnail.tsx: Made responsive with ResizeObserver, measures container size dynamically

**Learnings for future iterations:**
- The pixelation was caused by passing fixed canvas dimensions (240×60) but then stretching with w-full CSS class
- Canvas elements need their pixel dimensions to match their CSS display size, otherwise browsers stretch/blur the pixels
- For retina displays, multiply canvas width/height by devicePixelRatio (typically 2x) but cap at 2x to avoid performance issues
- ResizeObserver is the modern way to make canvas elements responsive - it triggers when container size changes
- The pattern: measure container → set canvas logical size → multiply by DPI → render → scale CSS to match logical size
- Don't mix fixed pixel widths with responsive CSS classes (w-full) - choose one approach
- The site-card component now controls layout (h-20), while thumbnail component handles rendering at appropriate resolution
